# Editorial Rules (Mandatory)

These rules apply to all current and future articles on the site.

## Core Ranking Rules
- Use E-E-A-T framing with practical test-based observations.
- Avoid generic filler and repetitive marketing language.
- Add specifics: numbers, scenarios, fit vs no-fit, and concrete use cases.
- Use plain US-style English. Keep phrasing simple and direct.

## Required Structure (Default)
- Summary / Verdict
- Who is it for
- Key features
- Pros & Cons
- Pricing
- Use cases
- Alternatives
- FAQ
- Final verdict

## Quality Standards
- Long-form depth: target 2000-4000 words per article type where applicable.
- Add internal links to other relevant guides/hubs.
- Include comparison tables when relevant.
- Include unique blocks such as:
  - Best for startups/agencies
  - Hidden drawbacks
  - Real use case
  - When NOT to use
- Add charts/visual benchmarks where appropriate.
